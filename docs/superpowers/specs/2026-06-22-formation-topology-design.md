# 球队阵型拓扑结构 V0 设计 (Formation Topology V0)

> 状态:设计已收敛,待评审 → 评审通过后出 TDD 实现计划(交 Codex)
> 日期:2026-06-22

## 0. 目标与边界

- **目标**:从转播视频的球员落点,算出可喂给 LLM 做战术解说的**结构信号**——核心是"落位防守 vs 高位逼抢"这类 high-level 战术态,以及 mentor 点名的**线间距**。
- **定位**:**纯位置后处理,0 GPU**。在 SoccerNet GSR 的 GT `bbox_pitch`(米制、瞬时)上开发验证,在标定输出(SoccerMaster / PnLCalib)上生产,**同一套逻辑**。等 GPU 的 SoccerMaster 跑通后直接换数据源即可,不阻塞。
- **交付物不是分类器**,是**可靠的数 + 一个可辩护的退路**。阈值 → 战术词的映射放在下游 prompt(由他人调研每种战术的距离区间填入)。

## 1. 锁定的决策

1. 锚点 = **A 滚动队形信号**(每隔几秒、在当前可见球员上算),不是离散阵型标签。
2. 范围 = **只刻画在场那一队**(镜头跟球 → 持球方进攻结构 + 失球方防守结构);永远不需要同时拿到 22 人。
3. V0 taxonomy = **位置可测的结构**;动态模式(逼抢触发 / gegenpress / 第三人跑动 / 套上内切 / 撞墙…)划出,归几何 / 事件模块。
4. 参考系 = 每队归一化进**规范进攻系**(永远朝 +x、本方门在 −52.5);几何与持球**解耦**——结构特征与持球无关,持球只作独立 tag。
5. 度量 = **足球四读 + 简版线**(见 §4),**不上聚类机器**(silhouette / Delaunay / role-based / EM 全部不进 V0)。
6. 线 = **按深度排序找最大间隙**;报**线间距**,**不报 "4-3-3" 具体线型计数**。

## 2. 输入契约

来自 sn-gamestate(GSR)+ tracklab 管线,每检测每帧:

- **落点**:`bbox_pitch.x_bottom_middle / y_bottom_middle` = 脚下点,米制。坐标系原点在中圈,x∈[−52.5, 52.5](门到门长轴),y∈[−34, 34](边线短轴)。标定失败该帧返回 `None`。
- `team` ∈ {`left`, `right`}(按**球门在画面哪一侧**定义,**非持球**),按 `track_id` 一次性判定。
- `role` ∈ {`player`, `goalkeeper`, `referee`, `ball`, `other`}。
- `jersey_number` int / NaN;**主键认 `track_id`,不认球衣号**。
- 预处理:仅 `role == 'player'` 进结构计算(GK / 裁判 / 球剔除);off-pitch 点已被 `remove_outside` 删;信单帧 team 前先按 track 整段确认。

集成口:本模块与已有的 `pitch_distances.py` **同级**,读同一个 `Labels-GameState.json` 的 `bbox_pitch`。

## 3. 规范化

- 上下半场各定一次进攻方向(由 team 标签给出,用门将 x 符号探测换边);把每队坐标翻成"朝 +x、本方门 −52.5"。
- 两队在同一窗各自算(各在自己的规范系里),互不依赖。

## 4. 度量(每队每窗)

### 免聚类四读(主)
- **block 高度** = outfielder 平均 x(规范系下离本方门多高)。低 → 落位,高 → 高位。
- **block 纵深** = `p90(x) − p10(x)`(抗离群)。小 → 紧凑,大 → 拉开。
- **block 宽度** = `p90(y) − p10(y)`。小 → 收窄,大 → 拉边。
- **三区占用** = 5 lane × 3 band 边际计数 + 左右 lane 不对称(球侧超载)。band 按规范 x 三等分;lane 按 y 五分(边路 / 肋部 / 中路 / 肋部 / 边路)。
- 辅:**凸包面积**(整体紧凑度)、**质心 y**(球侧偏移)、**最深 / 最高点 x**。

### 简版线(满足线间距需求)
- 把可见 outfielder 按 x 升序,取**最大的 1–2 个间隙**为线间断点 → 切成 2–3 条线。
- 报**线间距** D–M / M–F = 相邻两线的 x 质心差。
- **不报每线人数,不贴 "4-3-3"。**
- **退路**:可见人太少 / 最大间隙不显著(< Δ,Δ ≈ 6–8m)→ **不分线,退回三区占用**。

### 持球 tag(元数据,不进几何)
- 从最近持球人信号给每窗打 in / out-of-possession,供下游解读"进攻形 / 防守形"。出错只影响解读,**不污染上面的数**。

## 5. 时间窗(默认值,可调)

- 滑窗,**仅 in-play 帧**(死球 / 界外 / 定位球前摇剔除)。
- 每 track 取窗内**中位**落点(抗标定抖动)再算特征。
- 默认窗长 ~3s、步长 ~1s(给"及时"的战术态);需更稳的 settled shape 可叠一个长窗(~10–15s)。
- **覆盖门控**:可见 outfielder < N(暂定 8)→ 只出免聚类四读,简版线置空、`low_confidence=true`。

## 6. 输出

每窗每队一条记录(喂 prompt):

```json
{
  "t_start": 0.0, "t_end": 3.0, "team": "left", "possession_tag": "out",
  "block_height_m": -28.4, "block_depth_m": 18.2, "block_width_m": 31.5,
  "hull_area_m2": 410.0, "centroid_y_m": -2.1,
  "lane_counts": [2, 2, 3, 1, 2], "band_counts": [3, 4, 3], "side_overload": 0.18,
  "line_count": 3, "inter_line_gaps_m": [9.4, 12.1],
  "coverage_n": 9, "low_confidence": false
}
```

- 阈值 → 战术词("落位 / 高位 / 球侧压缩")由下游 prompt 填;V0 可附一个 rule-based 初判,仅供自检。

## 7. 验证

- 无阵型 GT → **对比验证**:从 NAS 世界杯整场截"一方持续落位"段 vs "一方高位组织"段,看 height / hull / 纵深是否分得开。
- 叠在已有 minimap 上肉眼核。
- **注意**:已传的 30s 是中圈开球段,settled block 没成形,**不能拿来测 low / high block**;开球段只适合看初始 nominal 快照。

## 8. 明确划出 V0(→ V1+)

动态模式(PPDA / 逼抢触发 / gegenpress / 第三人 / 套上内切 / 撞墙 / 身后球);"4-3-3" 等具体线型计数标签;silhouette / Delaunay / role-based / EM 等聚类机器。

## 9. 风险与假设

- `team_cluster` 外观聚类可能整条 track 误分 / 左右侧判反 → 污染该队结构。缓解:GK x 符号在半场上校验左右一致性。
- 部分可见使简版线在边卫缺席时偏(后四看成后二)→ 故**只报间距不报计数** + 覆盖门控 + 低置信标注。
- 标定 `None` 帧多的窗 → 覆盖不足 → 退三区占用。
- 规范系换边探测错 → 整队镜像翻 → height / 纵深全反。缓解:每半场各验一次。
